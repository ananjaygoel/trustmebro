Last semester we developed a text-to-SQL app. It was a simple one-agent architecture that takes a user query in natural language and after passing the validations checkpoint it generates a SQL query for it. (shocking, we know)

After completing the development we kicked off testing it and realised that one agent architecture was not enough for our app.

## The Details

It worked for simple queries where only 1 or 2 operations were required but it kicked off failing when the user kicked off asking complex queries, including multiple operations and schema scans. After thorough testing of the app we realised that a single agent couldnt perform every task.

The agent was trying to parse the intent, map it to a schema, generate a valid SQL, and validate its own output all in one go. Retry, the context was so bloated with flopped attempts and self-corrections that the agent kicked off contradicting itself.

## Why This Matters

After that we decided to revisit our architecture and implemented multi-agent system with specialised agents for every task instead of relying on just one agent to do everything.
